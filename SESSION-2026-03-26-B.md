## Architecture Highlights

### Call Abandonment System
- Webhook detects `caller-hung-up` + `status=ringing`
- Logs to dedicated analytics table
- Updates call record with `is_abandoned: true`
- Provides abandonment rate calculation per assistant

### Retry Logic Flow
1. Call ends with outcome (busy/voicemail/no-answer)
2. Webhook calls `scheduleRetry()`
3. Checks retry config (on_busy, on_voicemail, delay_minutes)
4. Calculates next attempt time
5. Updates campaign_contact with `next_attempt_at`

### Campaign Dashboard Architecture
- Real-time query aggregation (no caching)
- Activity log for event stream
- Metrics: active calls, call rate (calls/min), daily stats
- Suitable for live monitoring UI

### Timezone-Aware Dialing
1. Contacts have timezone field (default: America/New_York)
2. `getContactsByTimezone()` clusters pending contacts
3. `isCallingWindow()` checks if timezone is in business hours
4. Dialer selects next batch from appropriate timezone

## Performance Notes

- Abandonment tracking: O(1) - single insert
- Number health checks: O(n) where n = number of inbound numbers (recommend hourly cron)
- Campaign stats: O(m) where m = contacts in campaign (< 500ms for 1000 contacts)
- Timezone clustering: O(n log n) for sorting, acceptable for < 10K contacts
- Live dashboard: Aggregates in real-time, recommend caching for >100K calls
